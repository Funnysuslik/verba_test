Логика специально оставлена приземленной: один shop-файл + отдельные модули экспорта/фильтрации, без лишних слоев.
По хоршему должны быть обстракции с дефолтными настройками краулера и парсера, откоторых наслудется конкретный шоп.

## Что где лежит

- `shops/wb.py` — рабочий пайплайн: запрос -> парсинг -> выгрузка
- `src/loader.py` — XLSX-лоудер + мягкая валидация через `pydantic`
- `src/aggregator.py` — фильтр для второй выгрузки (рейтинг/цена/страна)

## Обязательные поля в выгрузке

Лоудер требует только:
- `product_url`
- `name`
- `sku`
- `price`

Остальные поля допускаются пустыми. Валидация нестрогая (`strict=False`), лишние ключи игнорируются.
Потому что они добываются доп запросами, без сессии прилетает бан. Сессию свою я банить спамом не хочу, спасибо.

## Запуск

```bash
python3 -m venv .venv
.venv/bin/pip install -e .
.venv/bin/python shops/wb.py
```

## Если WB отвечает нестабильно

- Первое, что смотрим: `headers` и `cookie` в `_build_search_request()`
- Второе: URL в `WBDefaults.BASE_URLS` (иногда один эндпоинт режется, второй живой)
- Третье: ключи в `parse_product_card()` и `_extract_products()`
